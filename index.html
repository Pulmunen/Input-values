Using event object's target property - multiple elements

This is much easier with event.target.value, because it allows for the event listener to be placed on an element in the hierarchy above the elements you want to listen to and react to events on any of those contained elements.

Submitting values by listening for a 'submit' event on a form - single and multiple elements

Instead of handling input values keystroke by keystroke, you can submit them on a submit or click event. If you use a form, then you can add a 'submit' event listener on the form itself. A 'submit' event listener on a form element will react to actions that fire submit events on its children/contained events. That means not just clicks on submit buttons but also pressing return key from an input field, etc.

I noticed that putting the event handler on the form to listen for events on child/contained elements it felt natural to switch from using an anonymous function to using a named function as the event handler's function argument.

Submitting values by listening for a 'click' event - single and multiple elements

Finally, if you're not using a form then you can submit input values from a div or section (or the document body etc.) by adding an event listener to a button element within the div or section to listen for a click event.
